# 🚀 Chapter 3-1. 소유권  
  
> 소유권은 Rust가 가비지컬렉터에 의존하지 않고도 메모리 안전성을 확보하려는 해법이다.
  
소유권은 Rust의 핵심 기능이다.  
보통 우리가 알고 있는 메모리 관리 기법은 C++처럼 직접 메모리를 관리해주는 방식,  
C#, Java, Go처럼 가비지컬렉터가 자동으로 메모리를 지속적으로 추적하며 메모리를 관리해주는 방식 이 두가지로 알고 있었다.  
  
하지만 Rust의 메모리 관리 방식은 이 두 가지 방식과 전혀 다르다.  
Rust의 메모리는 <u>컴파일러가 컴파일 시점에 검사하는 다양한 규칙으로 이루어진 소유권 시스템에 의해 관리</u>된다.
  
## 🔥 소유권 규칙  
  
소유권 시스템의 규칙은 다음과 같다.  
  
- Rust가 다루는 모든 값은 소유자(owner) 라고 불리는 변수를 가지고 있다.  
- 특정 시점에 값의 소유자는 단 하나뿐이다.  
- 소유자가 스코프(범위)를 벗어나면 그 값은 제거된다.  
  
여기서 스코프란, 변수가 유효한 범위를 일컫는다.  
  
```rust
fn main() {
    let a = 1;
}
```  
  
여기서 a는 main함수의 } 를 지나갈때 변수의 유효범위가 끝나는 것이다.  
  
## 🔥 String 타입  
  
String은 문자열을 다루는 타입이다.  
데이터 타입 챕터에서 설명을 안한 이유는 바로 이 소유권 시스템 때문이다.  
  
지금까지 알아본 데이터 타입들은 스택에 저장이 되었고, 그렇기에 모두 고정된 크기를 가졌다.  
하지만 String은 가변된 크기를 가졌으므로, 힙에 저장이 되는 데이터 타입이다.  
  
소유권은 힙 메모리를 관리하기 위해 생긴 시스템이다.
