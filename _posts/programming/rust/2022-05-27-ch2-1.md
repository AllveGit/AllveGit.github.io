# 🚀 Chapter 2-1. 변수와 가변성

> Rust의 특징은 C/C++와 동등한 속도를 달성하면서 동시성과 안정성까지 챙긴다는 것이다. 이 목표를 달성하기 위해 Rust에는 많은 제약과 규칙들이 있다.
  
그에 따른 규칙중 하나는 변수는 기본적으로 한번 초기화한 이후 <u>값을 바꾸지 못한다.</u>  
  
```rust
fn main() {
    let a = 1;
    println!("a: {}", a);

    a = 2; // 컴파일 에러!
    println!("a: {}", a);
}
```  
  
이런식으로 이미 선언된 a의 값을 바꿀려고 하면 컴파일 에러가 발생하게 된다.
  
즉, Rust의 변수들은 모두 <u>불변성</u>을 기본적으로 지니고 있다.
이 특성의 장점은 변수의 값이 언제 어디서 바뀌는지 일일이 추적할 필요가 없어진다.  

하지만, 변수는 변하는 값이라 변수이다. 즉, 변수에 가변성이 필요할 때가 더욱 많을 수 있다는 것이다.  

```rust
fn main() {
    let mut a = 1;
    println!("a: {}", a);

    a = 2; // 컴파일 성공!
    println!("a: {}", a);
}
```  
  
변수에 가변성을 추가해주기 위해서는 mut 키워드를 변수명 앞에 선언해주면 된다.  
이러면, 변수의 값을 초기화한 이후에도 변경할 수 있다.

## 🔥 변수와 상수의 차이

> 상수는 기본 불변이 아닌, 항상 불변이다.

```rust
const c1: u32 = 100_000;
const mut c2: u32 = 100_000; // 컴파일 에러!
```
  
let으로 선언한 변수들은 기본 속성이 불변인 것이지만, 상수는 mut 키워드를 붙일 수 없어 항상 불변이다.  
  
상수는 무조건 타입을 명시해주어야 한다. (변수: 타입) 포맷으로 지정할 수 있다.  
타입을 지정해주는 것을 <u>타입 애노테이션</u> 이라고 한다. 
  
상수는 상수 표현식으로만 초기화가 가능하며, 함수 호출 등의 런타임 연산으로 얻은 값을 상수로 초기화 할 수 없다.  
  
## 🔥 변수 가리기 (Shadowed) 
  
```rust
fn main() {
    let a = 1;

    let a = a + 1;

    let a = a + 2;

    println!("a의 값: {}", a);
}
```

let으로 이미 선언된 변수랑 똑같은 이름으로 다시 선언을 할때 변수가 가려졌다(Shadowed)고 한다.  
mut랑 똑같이 변수의 값을 바꾸는 것처럼 볼 수 있지만, 저기서 선언된 a들은 같은 객체들이 아니다.  
  
```rust
fn main() {
    let a = 1;

    let a = "test";

    println!("a의 값: {}", a);
}
```  
  
분명 첫번째 a는 정수형이지만, 두번째 a를 선언을 할 때는 문자열로 바뀌었다.  
그것은 변수가 가려지면 <u>원래 변수랑은 아무런 관계가 없이 새로운 변수로 탈바꿈</u> 이 되는 것을 의미하는 것이다.
