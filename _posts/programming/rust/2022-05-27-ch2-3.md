# 🚀 Chapter 2-3. 함수  
  
Rust는 변수와 함수의 이름에 스네이크 케이스(snake case)를 사용한다.  
스네이크 케이스란, 소문자와 밑줄 기호로 구분하는 형식이다.  
  
Rust에서 함수의 선언 순서는 중요하지 않으며, 어디에 선언했는지가 중요하다.  
  
## 🔥 함수 매개변수  
  
> Rust 함수의 매개변수에는 타입을 명시해주어야 한다.  
  
타입을 명시해주는 법은 타입 애노테이션을 사용한다.  
  
```rust
fn main() {
    func_test(100);
}

fn func_test(val: i32) {
    println!("{}", val);
}
```  
  
## 🔥 함수 본문의 구문과 표현식  
  
```rust
fn main() {
    let a = 10;
}  
```  
  
10이라는 값을 가지는 a라는 변수를 선언하는 간단한 코드이다.  
  
여기서 let 키워드를 이용해 변수를 생성하고 값을 대입하는 것은 구문이라고 한다.  
그리고 10이라는 값은 표현식이다. 10 그 자체의 값을 나타내므로 세미콜론을 쓰지 않는다.  
  
그렇다면 이 코드를 보자.  
```rust
fn main() {
    let a = 5;

    let b = {
        let a = 1;
        a + 1
    }
}
```  
  
이렇게 해도 값이 대입이 되나? 하는 코드일 것이다. 하지만 b에는 2라는 값이 정확히 들어간다.  
  
마지막에 세미콜론으로 끝나지 않는 a+1 문장이 있을것이다. 왜냐하면 10과 같이 표현식을 나타내기 때문이다. 그저 a+1 자체의 값을 나타내므로 b에 a+1의 값이 대입이 된다.  
  
만약 a+1에 세미콜론을 추가한다면, 컴파일 에러가 날것이다. 왜냐하면 a+1이 표현식이 아닌 구문으로 변환되어 리턴되지 않을 것이기 때문이다.  
  
## 🔥 리턴값이 있는 함수  
  
리턴값이 있는 함수도 마찬가지로 <u>리턴 값에는 세미콜론을 붙이지 않는다.</u>  
왜냐하면 함수를 이용하여 값을 받는 변수들이 원하는 것은 표현식이기 때문이다.  
  
```rust
fn func_test1() -> i32 {
    5
}

fn func_test2(val: i32) -> i32 {
    val + 1
}

fn main() {
    let a = func_test1();
    println!("a : {}", a);

    let b = func_test2(1);
    println!("b : {}", b);
}
```
