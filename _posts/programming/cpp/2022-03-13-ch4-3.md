# 🚀 Chapter 4-3 : 메모리와 이름  
  
> 메모리를 다룰 때는 해당 메모리 블록에 이름을 붙여서 사용한다. 왜냐하면 이름을 붙여서 사용하는 것이 가장 편하기 때문이다.

C++은 메모리에 접근하기 위하여 포인터를 사용한다. 포인터는 메모리를 좀 더 쉽게 다룰 수 있는 하나의 수단이기에 <u>포인터가 없더라도 메모리에 접근할 수 있는 방법은 있다.</u>

## 🔥 l-value로 사용되는 이름

> 일부 메모리 블록의 이름은 l-value(좌측 값)로 사용될 수 있다.

프로그래밍에는 l-value(좌측 값), r-value(우측 값)이라는 개념이 있다.    
대입 (=) 연산자를 기준으로 왼쪽 값, 오른쪽 값의 유래로 용어가 지어졌다.

<u>모든 C++의 식은 l-value 또는 r-value이다.</u>  
  
l-value는 유지되며 지속되는 개체이며,  
r-value는 유지되지 않는 임시적인 값을 나타낸다.  

🔔  
( ex. int a = 1; )
a는 l-value이고, 1은 r-value이다.
{: .notice}  

간단히 모든 변수는 l-value로 생각하면 된다.  
  
```c++
class Test
{
public:
  int m1;
  int m2;
};

void main()
{
  int a = 1;      // (1)
  int* pA = &a;   // (2)

  Test b;
  b.m1 = 1;       // (3)
  b.m2 = 2;       // (4)
}
```

위 코드는 l-value로 메모리 블록의 이름을 l-value로 쓰는 예제코드이다.  
  
a와 pA, b 모두 각각 메모리 블록을 나타내고 있으며 그 값을 설정할 수 있다.  
<u>즉, l-value는 값이 설정될 수 있는 메모리 블록을 의미한다.</u>  
  
<u>상수들은 l-value가 아니다.</u> 왜냐하면, 상수가 메모리 영역을 차지하는 경우도 있지만, 상수에다가 값을 설정할 수 없으며 상수 자체가 주소를 나타내지는 않기 때문이다.  
  
그렇지만 <u>const를 붙인 변수는 l-value이다.</u> 왜냐하면, 변수는 분명히 메모리 블록을 나타내며 최초로는 값을 설정할 수 있기 때문이다. 단지 최초 1회를 제외하고 값 설정이 금지된 l-value이다.  
  
<u>배열의 이름은 l-value가 아니다.</u> 메모리 블록을 나타내긴 하지만 배열의 이름으로는 값을 설정할 수 없기 때문이다.  
마찬가지로, <u>함수의 이름도 l-value가 아니다.</u>  
  
🔔  
직접 참조(.) 연산자는 메모리 블록의 부분 영역을 가리키기 위하여 사용한다.
{: .notice}  
  
## 🔥 r-value로 사용되는 이름(값)

> l-value와 마찬가지로 메모리 블록의 이름 자체를 r-value 값으로도 사용할 수 있다.

```c++
void main()
{
  int a = 1;
  int b = a;

  int* pA = &a;
  int* pB = pA;
}
```  

이렇게 메모리 블록의 이름을 r-value로도 사용이 가능하다.  
<u>여기서 의미하는 r-value는 메모리 블록의 비트 상태가 나타내는 값이다.</u>

## 🔥 r-value로 사용되는 이름(주소)

> 배열 이름과 함수 이름은 l-value로 사용되지 않으며, r-value로만 사용이 된다.  
이 때, 나타내는 값은 메모리 블록의 주소이다.

메모리 블록의 이름이 r-value로 쓰일 때는 메모리 블록의 비트 상태가 나타내는 값을 의미한다고 하였다.  
하지만, 다른 의미로 쓰이는 r-value들도 있다. 바로 배열 이름과 함수의 이름이다.

<u>배열과 함수 이름은 메모리 블록의 주소를 나타낸다.</u>  
  
하지만, 배열은 초기화 식을 넣을 수 있으므로 l-value도 되는 것이 아닌가?  

```c++
void main()
{
  int arr1[1] = {1};
  int arr2[1];

  arr2 = arr1; // Compile Error!

  int* pArr = arr1;
}
```  
  
예제코드를 보면 arr1에 값을 대입하고 있으므로 arr1은 l-value인 것 같다.  

기존 l-value의 값 설정 방식은 메모리 블록을 마련한 뒤 이름을 붙여서 그 이름을 통해 값을 대입하였다.  
하지만, <u>배열 초기화는 해당 메모리 블록을 마련하고 1이라는 값을 채워넣은 다음, arr1이라는 이름을 붙인다.</u>  
  
결정적으로, arr2에 arr1을 대입을 못하기 때문에 배열 객체는 l-value로 사용할 수 없는 것이 결론이 된다.
  
그리고 pArr에 arr1이 대입이 되는 케이스를 보면 배열의 이름은 메모리 블록의 주소를 나타내는 것도 확인할 수 있다.  
  
함수의 이름 케이스도 알아보자.  
  
```c++
void Func() {};

class Test
{
public:
  void MemberFunc() {}
};

typedef void (Test::*PFUNC) ();

void main()
{
  if (Func == &Func)            // (1)
  {
    cout << "Same!" << endl;    
  }

  PFUNC p1 = Test::MemberFunc;  // (2) Compile Error!
  PFUNC p2 = &Test::MemberFunc;
}
```

예제코드를 보면 (1)을 통해 Func는 함수 이름이기 때문에 주소 연산자(&)을 사용하여 비교 할 경우 똑같다는 사실을 알 수 있다.  
  
즉, <u>함수의 이름은 함수본체의 메모리 주소를 나타낸다.</u>  
  
하지만 <u>멤버 함수는 함수 이름이 주소를 나타내는 r-value로 쓰이지 못한다.</u>  
(2)에서 함수 포인터 변수가 멤버 함수 이름으로 대입이 안되기 때문이다. 멤버 함수 이름 앞에 &을 붙이면 정상 작동하는 것을 확인 할 수 있다.  
  
멤버 함수 이름도 함수본체의 메모리 주소를 나타낸다. 하지만, <u>컴파일러가 강제로 &연산자를 붙이도록 강제했을 뿐이다.</u>   
(2)처럼 표기해도 주소를 나타내면 함수 이름뒤에 괄호가 붙을 경우 함수 호출을 해야하는지 함수 주소를 리턴해야하는지 기준이 모호해지기 때문이다.
