# 🚀 Chapter 5-3 : 다차원 배열

> 배열의 요소 타입은 모든 타입이 올 수 있다. 즉, 배열의 요소 타입으로 배열 타입이 오는 것이 가능하다는 의미이다.  
  
즉, 배열 안에 배열을 집어넣음으로써, 다차원 배열을 구현할 수 있다.

## 🔥 2차원 배열 구조

```c++
void main()
{
  int arr[2][3] = { {11, 12, 13}, {21, 22, 23} }
}
```

위 예제코드는 2차원 배열을 생성 및 초기화한 코드이다.  
의미는 int 객체 3개가 모인 배열 객체를 2개 모아놓은 것이다.    

❗  
int 객체 2개가 모인 배열 객체를 3개 모아놓았다고 해석하면 안된다!  
순서는 항상 올바르게 인지하고 있어야 한다.  
뒷 배열요소 개수가 모인 배열 객체를 앞 배열요소 개수만큼 모아놓았다고 해석해야 한다.
{: .notice--warning}  
  
이런 다차원 배열을 행렬(Matrix) 와 같은 구조로 생각할수도 있다.  
하지만 프로그래밍에서 다루는 메모리, 즉 가상 메모리는 일렬로 쭉 나열된 선형적인 구조를 지니고 있다.  
  
그렇기에, 메모리 적재 구조상 행렬같은 구조로 다차원 배열을 표현할 수 없다.  
<u>다차원 배열은 몇차원이던간에 무조건 선형으로 요소가 이어진다.</u>  
  
<!-- 2차원 배열 메모리 적재 구조 이미지 삽입 필요 -->  
  
배열 이름은 무조건 r-value로만 쓰일 수 있다고 하였는데,  
예제 코드에서의 <u>arr[0]과 arr[1]도 마찬가지로 배열을 나타내므로 r-value로만 사용될 수 있다.</u>  

## 🔥 포인터 대응
  
> 포인터에서 첨자 연산자([])를 사용하면 배열처럼 사용할 수 있다.  
하지만 포인터와 배열은 서로 다른 구조를 가지고 있다는 걸 명심해야 한다.  
  
```c++
char s1[] = "ABC";
char* s2 = "ABC";
```  
  
<!-- 배열과 포인터의 메모리 구조 차이 이미지 삽입 필요 -->

s1과 s2의 구조는 아예 다르다.  
  
둘의 같은 점은 둘 다 문자열이 들어있는 메모리 블록의 주소를 나타내는 것뿐이다.  
  
포인터와 배열은 다르다는 것을 알 수 있을 것이다.  
왜냐하면, <u>포인터는 l-value로도 쓸 수 있지만, 배열은 r-value로만 쓸 수 있기 때문이다.</u>  
포인터는 간접 연산자(*)를 앞에 붙이면 원천타입의 크기만큼의 메모리 블록만 나타낸다.  
  
그런데 지금까지 포인터에 첨자 연산자를 사용하는 것과, 배열을 사용하는 것의 차이가 없었다.  
그러면 포인터&첨자연산자를 배열과 똑같이 생각해도 되지 않을까?  
  
<u>배열과 포인터를 편하게 호환해서 사용할 수 있는 것은 1차원 배열인 경우뿐이다.</u>  
  
```c++
void main()
{
  int arr[2][3] = { {11, 12, 13}, {21, 22, 23} };
  int** ppArr = (int**)arr;
  cout << ppArr[1][2] << endl;
}
```  
  
이 결과는 23일까? 결과는 예외가 발생하거나 쓰레기 값이 출력될 것이다.  

이 예제의 동작과정은 시스템 플랫폼(x86, x64)에 따라 달라진다.  
ppArr의 원천타입은 int* 인데 이것은 플랫폼에 따라 크기가 달라지기 떄문이다.  
x64는 int는 4바이트인데 int*는 8바이트이다. 이로 인해 뭔가 예외가 발생할 것이 예상이 간다.  
  
<!-- x86 시스템 2차원 배열 구조 이미지 삽입 필요 -->    

예제코드가 x86 시스템에서 동작하는 과정을 담은 이미지이다.  
  
ppArr[1] 에서 가리키는 값은 12이다.  
그런데 ppArr의 원천타입은 int* 이므로 12라는 값을 int* 로 해석하게 된다.  
그러므로, ppArr[1]은 12라는 메모리 주소를 나타내는 포인터가 된다.  

ppArr[1]은 int* 타입이니 원천타입은 int가 된다. 따라서 첨자 연산자를 사용하면
int의 크기인 4바이트만큼씩 메모리 주소를 이동한 후에 메모리의 값을 가져온다.  
그러므로, ppArr[1][2]는 메모리 주소 20 (12에 4바이트 * 2를 더함) 의 값을 가져온다.  
  
하지만, 메모리 주소 20은 값이 정의가 되어있지 않기에 쓰레기 값이 출력되거나  
접근 가능한 메모리 주소가 아닐수도 있으므로 메모리 예외가 발생할수도 있다.   

<!-- x64 시스템 2차원 배열 구조 이미지 삽입 필요 -->

예제코드가 x64 시스템에서 동작하는 과정을 담은 이미지이다.  
  
x64 시스템에서 포인터의 크기는 8바이트이다. 하지만 int의 크기는 그대로 4바이트이다.  
그렇기에, ppArr의 원천타입은 int* 이므로 ppArr[1]은 arr[0][2]과 arr[1][0]을 둘다 차지하는 영역을 가진다. 이렇게 두 개의 영역을 차지했을 때는 두 메모리 영역의 값 바이트들을 이어붙인(더하는 것이 아님) 값을 int*로 해석하게 된다.  
  
낮은 주소 4바이트 arr[0][2]의 값은 13이며, 16진수로 0x0000000D이다.  
높은 주소 4바이트 arr[1][0]은 값은 21이며, 16진수로 0x00000015이다.  
  
이 두 주소를 이어붙이면 8바이트 16진수 0x150000000D 가 만들어지게 된다.  
이것을 int*로 해석하면 ppArr[1]은 0x150000000D라는 메모리 주소를 나타내는 포인터가 된다.

ppArr[1]은 int* 타입이니 원천타입은 int가 된다. 따라서 첨자 연산자를 사용하면
int의 크기인 4바이트만큼씩 메모리 주소를 이동한 후에 메모리의 값을 가져온다.  
그러므로, ppArr[1][2]는 메모리 주소 0x1500000015 (0x150000000D에 4바이트 * 2를 더함) 의 값을 가져온다.  

마찬가지로, 메모리 주소 0x1500000015 값이 정의가 되어있지 않기에 쓰레기 값이 출력되거나  
접근 가능한 메모리 주소가 아닐수도 있으므로 메모리 예외가 발생할수도 있다.  
  
그렇다면 2차원이상의 배열은 포인터와 호환될 수 없는 것일까?  
  
```c++
void main()
{
  int arr[2][3] = { {11, 12, 13}, {21, 22, 23} };

  int* arrP[2] = {arr[0], arr[1]};
  int** ppArr = (int**)arrP;

  cout << ppArr[1][2] << endl;
}
```  
  
이 예제코드는 ppArr을 int* 타입 포인터 두개를 가지는 배열 객체 arrP를 가리킨다.

<!-- 2차원 배열 해결법 구조 이미지 삽입 필요 -->

이렇게 되면 ppArrP[1]은 arrP[1]과 똑같은 의미를 가지게 된다.  
그렇기에, ppArrP[1]은 arr[1]의 시작주소를 가리키는 포인터를 의미한다.  
ppArrP[1]의 원천타입은 int이므로 첨자 연산자를 사용해 4바이트(int의 크기)씩 메모리 주소를 이동해 값을 정상적으로 가져올 수 있다.  
  
왜 해결이 되었을까?  
그것은 ppArr[0], ppArr[1]이 정상적인 메모리 주소를 가리킬 수 있도록  
arrP[2]를 만들어 각각의 메모리 시작주소를 설정한 후, ppArr로 변환시켰기 때문이다.

## 🔥 배열 전달

> 배열을 인자로 전달할 때 포인터를 많이 사용한다. 하지만, 포인터가 아닌 배열 자체를 인자로 넘기는 법도 있다.  
  
```c++
int PointerFunc(int* arg)
{
  return sizeof(arg);
}

int Pointer2Func(int arg[])
{
  return sizeof(arg);
}

void main()
{
  int arr[4] = {0};
  int pointerSize = PointerFunc(arr);
  int pointer2Size = Pointer2Func(arr);

  cout << "arr size : " << sizeof(arr) << endl;
  cout << "pointer arg size : " << pointerSize << endl;
  cout << "pointer2 arg size : " << pointer2Size << endl;
}
```  
  
예제 코드를 보면 int* arg, int arg[] 방식으로 각각 함수 호출 후 인자의 사이즈들을 리턴해준다.  
  
예제 코드의 결과는 x86 시스템에서 [ 16, 4, 4 ]가 나온다.  
4바이트가 나오는 이유는 포인터이기 때문이다.  
int arg[]로 넘기는 방식도 결국에는 int* arg랑 똑같은 포인터 취급을 한다는 의미이다.  
  
그렇다면 배열 자체를 전달하는 방식은 없을까?  
  
```c++
int RefFunc(int (&arg)[4]) // Array Reference Type 
{
  return sizeof(arg);
}

void main()
{
  int arr[4] = {0};
  int refSize = RefFunc(arr);

  cout << "arr size : " << sizeof(arr) << endl;
  cout << "ref arg size : " << refSize << endl;
}
```  
  
배열 참조 타입으로 인자를 전달하면 배열 자체가 전달이 된다.  
이렇게 전달하면 우리가 원한대로 배열 사이즈 그대로 [ 16, 16 ] 이 결과로 출력된다.
