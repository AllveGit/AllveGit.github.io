# 🚀 Chapter 4-1 : 가상 메모리

> 가상 메모리를 실제 메모리보다 메모리를 많이 쓸 수 있도록 가상으로 늘려주는 기술 정도로 알고 있는 사람들이 있다. 그렇게 알고 있다면 확실히 개념을 파악하자.

<u>가상 메모리의 핵심은 각각의 프로세스에게 메모리 공간이 독립적으로 부여</u>된다는 것이다.  
  
메모리 영역이 프로세스에게 독립적으로 부여되기 때문에 타 프로세스가 해당 메모리 영역에 침범할 수 없게 된다.  
  
그래서 가상 메모리를 사용하면 프로그램의 안정성이 크게 향상된다.

## 🔥 가상 메모리 크기

> 가상 메모리의 크기는 시스템 플랫폼에 따라 달라진다.

32비트 시스템(x86)에서는 <u>각각의 프로세스가 4GB의 가상 메모리를 부여</u>받는다.  
   
왜 4GB일까?  
32비트 시스템이라는 것은 메모리 주소를 가리킬 수 있는 레지스터의 크기가 32비트라는 것이다.  
그러니 2의 32승인 대략 42억 개의 주소를 사용할 수 있고 주소는 1바이트씩 차지하므로 4GB이다.  
  
하지만, 이 4GB에서 <u>절반은 운영체제가 사용하기 때문에</u> 사실상 프로세스에서 사용할 수 있는 메모리 공간은 2GB가 된다.  
  
64비트 시스템(x64)에서는 <u>윈도우에서는 각각의 프로세스가 8TB의 가상 메모리를 부여</u>받는다.    
  
왜 8TB일까? 마찬가지로 2의 64승개의 주소를 사용할 수 있기에 대략 1천 6백만 TB 가상 메모리 공간이 부여될 수 있을 것이다.  
  
하지만, 그만큼 큰 메모리 공간이 필요한 프로그램은 존재하지 않기에,  
<u>윈도우는 대략 8TB 공간만을 부여하도록 제한</u>하였다.  
마찬가지로, 절반은 운영체제가 사용하므로 4TB를 프로세스에서 사용 가능하다.  

## 🔥 가상 메모리 주소  
  
실제로 프로세스들은 메모리에 올라가서 작동이 된다. 그렇지만, RAM 메모리의 크기는 크지 않은데 기가바이트 단위의 프로세스들을 어떻게 여러 개를 한꺼번에 실행시킬 수 있는걸까?  
  
가상 메모리는, 가상의 메모리를 만드는 것이 아닌 <u>SSD나 HDD같은 하드디스크들을 RAM처럼 사용</u>하여 공간을 늘리는 것이다. 이 가상 메모리에 프로세스를 적재한 후, 프로세스에서 <u>필요한 부분이 있다면 그 부분만 물리 메모리로 적재</u>시켜준다. 이것을 swap이라는 용어로 표현한다. 그렇기에 물리 메모리는 가상 메모리에 의해 계속 swapping을 한다.  
  
프로세스들은 이런 장점과 더불어 서로의 메모리 영역을 침범하지 않기 위해 가상 메모리 주소를 사용해야 한다. 즉, 프로세스는 <u>가상 메모리 주소들을 사용하지만 데이터에 접근할 때만 물리 메모리 주소로 변환</u>하여 접근하는 것이다.  
  
가상 메모리 주소를 알기 전에 알아야 하는 상식이 있다.  
나중에 OS를 정리할 일이 있으면 자세하게 하겠지만 일단은 간략하게 정리해보겠다.  
  
OS에서는 메모리가 낭비되는 메모리 단편화라는 현상이 있는데, 이것을 줄이기 위해  
페이징(Paging)이라는 것을 통해 가상 메모리와 물리 메모리들을 동일한 크기로 자른다.  
이 자른 메모리들을 가상 메모리에서는 페이지(page), 물리 메모리에서는 페이지 프레임(pageframe) 이라고 부른다.  
  
🔔  
일반적으로 페이지는 4KB 메모리 덩어리를 기준으로 한다.

![Fundamental-Cpp-BlogImages-6](https://user-images.githubusercontent.com/39856790/159162505-98282abe-bc4a-4b59-bf6e-32af5643810f.jpg)  
  
각 프로세스는 페이지 테이블을 가지고 있다.  
페이지 테이블은 가상 메모리의 각 페이지가 물리 메모리의 어떤 페이지 프레임과 연결 되어있는지가 기록된다.  
  
> 즉, 프로세스들은 가상 메모리 주소로 페이지 테이블의 정보를 이용해 물리 메모리 주소로 변환해 데이터를 읽고 쓸 수 있다. 자세히 설명하면, MMU라는 하드웨어를 통해 변환하는 것을 알 수 있는데 이건 추후에 설명할 기회가 있으면 하도록 하겠다.

## 🔥 가상 메모리 구조

> 프로세스는 오직 가상 메모리만을 바라본다. 즉, 물리 메모리에 대해서는 신경을 쓰지 않고 운영체제에 맡기게 된다. 그렇기에 C++에서의 메모리는 프로세스의 가상 메모리를 의미하는 것이다.  
  
![Fundamental-Cpp-BlogImages-7](https://user-images.githubusercontent.com/39856790/159162816-b056db22-c83b-4bdc-a886-84347c7bcc31.jpg)  
  
이 이미지는 운영체제가 사용하는 메모리 영역을 제외한 프로세스가 사용할 수 있는 메모리 영역의 구조 이미지이다.  
  
- Code(Text) 영역
  * 이 영역은 소스코드를 컴파일시 생성되는 기계어가 저장이 된다.  
  * 이 영역의 데이터는 실행코드 변조 방지를 위해 쓰기가 금지되며, 오직 읽기만 가능하다.  
    
- Data & BSS 영역
  * 이 영역들에는 전역 변수와 전역 배열이 저장이 된다.
  * 데이터 영역은 초기화된 전역 객체가 저장되는 영역이다.
  * BSS 영역은 초기화되지 않은 전역 객체가 저장되는 영역이다.  
    
🔔  
일반적으로 페이지는 4KB 메모리 덩어리를 기준으로 한다.
{: .notice}  

- Heap 영역
  * 이 영역은 동적 메모리가 저장이 된다.
  * 프로세스마다 하나 이상도 있을수도 있다.
  * 가상 메모리로부터 큰 메모리 덩어리를 할당 받은 후, new 같은 요청이 올 때 이 할당받은 메모리 영역을 분할하여 전달한다.  
  * 힙 관리자에 의해 관리된다.


- Stack 영역
  * 이 영역은 함수의 인자, 지역 변수, 지역 배열 등이 저장된다.
  * 이 영역은 스레드당 하나씩 생성된다. (컴파일러 설정에 따라 1~4MB 정도로 할당됨)
  * 스레드가 생성되는 순간 운영체제가 가상 메모리의 남은 영역중 적당한 것을 골라 할당해줌.
