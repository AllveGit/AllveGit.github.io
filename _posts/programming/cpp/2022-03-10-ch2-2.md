# 🚀 Chapter 2-2 : 기억부류 (Storage Class)
  
## 🔥 논리 공간

컴파일러는 소스파일 하나씩 목적(.obj) 파일을 생성하게 된다.
이 목적파일들이 링크되어서 하나의 실행파일(.exe)를 만들게 된다.

.obj의 공간을 오브젝트 공간, 실행파일의 공간을 실행 공간이라 정의하겠다.

- 같은 이름의 전역 변수는 실행공간에 단 한번만 정의될 수 있다.  
- 같은 이름의 정적(static) 변수는 각각의 오브젝트 공간에 단 한번만 정의될 수 있다.
  (실행 공간에는 여러 번 정의 될 수 있지만, 각각 다 다른 개별적인 변수이다.)
- 같은 이름, 같은 인자 타입을 가진 전역 함수는 오브젝트 공간에 단 한번만 정의될 수 있다.
- 같은 이름, 같은 인자타입, 같은 반환타입을 가진 함수는 실행 공간에 단 한번만 정의될 수 있다.
- 같은 이름의 클래스는 각각의 오브젝트 공간에 단 한번만 정의될 수 있다.
  
## 🔥 변수 할당과 초기화
  
> 변수는 정의할 때 메모리를 할당하면서 초기화 하는 것처럼 보일 수 있지만,  
엄밀히 순서차이는 있다.

정적 변수를 포함한 전역 변수는 프로세스가 시작될 때 메모리를 할당받게 된다.

```c++
/////////////////////////////////////
// A.cpp

int g_ValA;

/////////////////////////////////////
// B.cpp

int g_ValB;
```

이러면 어떤 변수가 먼저 메모리 할당을 받게 될까?

정답은 알 수 없다. 가상 메모리중에서 코드와 데이터 영역은 정해진 순서를 가지고  
메모리로 올라오는 것이 아니기 때문이다.

지역 변수는 함수 실행 시점에 지역 변수들의 크기를 전부 구해 스택 메모리의 크기를 늘린다.

즉, 전역/정적, 지역 객체의 메모리 할당 순서는 별로 중요하지 않다.  
하지만, 초기화의 경우는 순서가 중요하다.

```c++
/////////////////////////////////////
// Main.cpp

int Func(int arg)
{
    return arg;
}

int g_Val1 = Func(1);
int g_Val2 = 2;
```

이 예제에서 할당이 끝난 이후 초기화를 하면 어떤 순서로 초기화하게 될까?

**정답은 g_Val2, g_Val1 순서로 초기화하게 된다.**

코드는 위에서 아래로 실행되는 것이 규칙이다. 그런데 왜 g_val2가 먼저 초기화가 되는건가..

그 이유는 간단하다. 
프로세스가 시작될 때는 최초에 호출되어야 하는 시작 함수, CRT Startup이 호출 된 다음에 main이 호출되기 때문이다. g_val2는 main이 호출되기전에 이미 초기화가 된 상태라는 것이다. 

프로세스가 실행된다는 것은 스레드를 실행한다는 것이며, 스레드가 실행하는 것은 바로 함수이다.

엄밀히 말하자면 g_Val1은 CRT Startup이 호출되기전에 0으로 설정되어있다. 즉, 스레드를 통해 Func를 호출하여야 초기화가 될 것이며, 최소한 그 호출 시점은 CRT Startup이 호출된 다음이다. 

그럼 왜 g_val2는 먼저 초기화가 되어있는 상태일까? 그것은 g_val2가 함수가 아닌 상수로 초기화하기 때문이다. 전역변수의 초기화값이 상수인 경우에는 컴파일러에 의해서 데이터 영역에 직접 값을 설정할 수 있으며, 그 값들이 그대로 가상메모리에 올라오기 때문이다. 초기화하지 않은 전역변수들은 그 변수들을 위한 영역 BSS 영역이 있다. 그 영역에서 가상 메모리로 올라올때 0(NULL)로 초기화가 이루어진다.

지역 변수의 초기화는 순서대로 잘 이루어진다. 왜냐하면, 지역변수는 함수 안에 있으므로 함수가 호출된 것이기에 CRT Startup이 이미 호출된 다음이기 때문이다."

## 🔥 변수 초깃값, BSS 영역
  
변수가 정의되어 생성될 때 변수의 초깃값은 무엇일까?
변수의 메모리 영역의 모든 bit를 0으로 만들면 될까?

아쉽게도, 모든 변수에 대해 이런식으로 초기화를 하지 않는다.
왜냐하면 초기화를 하는데에도 CPU가 사용되기 때문이다.

```c++
void main()
{
    int a, b;
    a = 1;
    b = 2;
}
```

이런 식의 코드가 있다면, 굳이 a와 b에 0으로 초기화해야 할 이유가 있을까?

그래서 C++은 지역변수 및 힙변수들은 초기화하지 않고, 전역변수들만 초기화하도록 설계하였다.
왜냐하면, 지역변수들은 함수가 호출될 때마다 초기화하여야 하지만, 전역변수들은 프로세스 시작순간 단 한번만 초기화하면 되기 때문이다.

그래서 C++은 초기화하지 않은 전역변수들을 초기화하기 위해 BSS영역이라는 것을 설계하였다. 변수가 생성되는 시점에 초깃값을 가지지 않으면 안전하지 않기 때문이다.
BSS 영역은 최상의 효율을 내기 위해 초기화되지 않은 전역변수들을 가상 메모리 한쪽에 전부 몰아넣고 그 영역 전체를 0으로 채워 한번에 초기화를 한다.

```c++
int Func()
{
    return 1;
}

int g_A;
int g_B = 0;
int g_C = Func();

int g_D = 1;
```

여기서 g_A~g_C는 모두 BSS영역에 포함된다. g_C가 포함되는 이유는 함수가 실행되기전에 초깃값이 없으므로 초기화를 시키기 때문이다. g_D는 0이 아닌 상수로 초기화하기에 BSS영역에 포함되지 않는다. 

실제로 g_A~g_C를 &를 붙여 주소를 출력해보면 세 변수의 주소는 가까이 있지만,
g_D의 주소는 멀리 떨어져있는 결과를 볼 수 있다.

BSS영역으로 전역변수를 초기화하면 메모리 영역만 0으로 바꿔주면 되는 영역이기 때문에 최종 바이너리 크기에도 거의 영향을 미치지 않는다.

하지만, 0이 아닌 상수로 초기화하는 전역변수들은 최종 바이너리 크기에 값들의 크기만큼 영향을 준다.

## 🔥 디버그 변수 초깃값
  
변수들을 생성하여도 초기화가 되지 않는다면 디버그를 하는 시점에서 확인하기 어려울 것 같다.

그래서 vs는 디버그모드에서 지역변수와, 힙 영역을 특정한 값으로 초기화를 해준다.

- 지역변수는 16진수로 **cccccccc** 값으로 초기화 된다.
- 힙 할당 메모리는 16진수로 **cdcdcdcd** 값으로 초기화 된다.
- 힙 할당 메모리의 양끝은 16진수로 **fdfdfdfd** 값으로 초기화 된다.
- 해제된 힙 할당 메모리는 16진수로 **feeefeee** 값으로 초기화 된다.
