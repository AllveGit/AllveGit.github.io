# 🚀 Chapter 5-1 : 포인터

> 포인터는 메모리 블록의 주소를 가리키는 객체를 의미한다.  
  
하지만, 포인터가 메모리 블록의 주소만 가리킨다면, 값을 가져오거나 쓰고자 할 경우에 블록의 타입과 크기를 고려하며 코딩을 해야하는 불편함이 있을 것이다.  
  
그래서 <u>포인터는 주소뿐만이 아니라 타입 정보까지 기록한다.</u> 그렇기에 포인터는 타입 정보를 통하여 타입 크기만큼 메모리 블록에 값을 읽거나 쓸 수 있다.

## 🔥 포인터 타입

int 타입의 객체 메모리를 나타내기 위해서는 int* 포인터를 사용한다. 이때, int* 의 원천타입은 int이다.
int* 를 사용하는 것은 사용하는 것이 쉬워지고 편해지기 때문에 사용하는 것이지,  
무조건 int* 를 사용해야 되는것은 아니다. 즉, 다른 타입의 포인터를 이용해서 int를 가리킬수도 있다는 말이다.  

```c++
void main()
{
    int a= 0;
    int* pA = &a;

    *pA = 1;
}
```

이 예제 코드는 포인터의 원천타입과 객체의 타입이 일치하는 경우이다.  
포인터의 원천타입과 가리키는 객체의 타입이 같으므로, pA에 값을 쓰면 객체의 타입 크기(4byte)만큼의 값을 쓴다.  
이 때, <u>값을 쓸 때 int 타입에서 1이라는 값을 가지도록 타입에 대응하여 비트 상태를 바꾼다.</u>  
  
```c++
void main()
{
    int a = 0;
    char* s = &a; // (1) Compile Error!
    char* s = (char*)&a;

    *(s + 0) = 'A';
    *(s + 1) = 'B';
    *(s + 2) = 'C';
    *(s + 3) = '\0';
}
```

이번 예제 코드는 포인터의 원천타입과 객체의 타입이 불일치하는 경우이다.  

일단 int a = 0; 에서 a가 가리키는 메모리블록 4바이트의 비트상태는 모두 0이 될 것이다.  

(1)번 케이스에 보듯이 포인터가 원천타입이 다른 객체의 주소를 받으면 컴파일 에러가 난다. 왜냐하면, 컴파일러가 포인터의 원천타입과 똑같은 타입의 객체의 주소를 받는 것을 권장하기 때문에 거부한다.  
하지만 강제변환을 통해 포인터는 원천타입과 다른 객체의 주소를 나타낼 수 있다.  
  
char* s의 원천타입은 char이므로 1바이트씩 데이터를 읽고 쓸 수 있다.  
(s + 0), (s + 1), (s + N)... 은 해당 바이트만큼 메모리 주소를 이동한다는 의미가 아닌 sizeof(char) 크기의 배수만큼씩 이동한다는 의미이다.  

실제로는 int로 메모리를 할당받았으므로, 4바이트 메모리 블록을 1바이트씩 4번 char로 데이터를 쓰는 것을 보여준다.

## 🔥 타입 표기

> 포인터를 표기할 때에는 TYPE* 으로 표기가 된다. 하지만, TYPE* 방식으로 표기 못하는 타입들도 있다. 다른 방식으로 표기를 해야되기에 포인터 표기 방식엔 여러가지가 있다.  
  
```c++
int arr[2] = {1, 2};
```

이때 arr을 포인터로 어떻게 표현할까? arr의 타입은 무엇일까?  
변수 이름을 제외하고 보면 int[2]가 아닐까? 그렇지만, int[2] arr은 성립되지 않는다.  
  
그렇기에 TYPE* 방식으로는 배열 arr의 타입을 나타낼 수 없다.  
이 같은 타입을 표현할 수 있는 다른 표기법들을 알아보자.

```c++
typedef int (TARR) [2]
typedef int (*PARR) [2];

int arr[2] = {1, 2};

void main()
{
    TARR* p1 = &arr;
    PARR p2 = &arr;
}
```

첫번째 다른 표기법이다. typedef를 이용해 타입을 정의하는 방식이다.  
   
TARR은 typedef를 이용해 arr의 타입을 나타내었다.  
그렇기에 TARR에 포인터를 붙여주면 arr을 포인터로 표현할 수 있다.  
  
PARR은 타입을 정의할 때부터 arr타입의 포인터로 정의를 한다.  
그렇기에 arr을 포인터로 표현할 수 있다.
  
```c++
int arr[2] = {1, 2};

template<typename T>
void ArrayPrint(T InArg)
{
    cout << (*InArg)[0] << (*InArg)[1] << endl;
}

void main()
{
    ArrayPrint<int (*)[2]>(&arr);
}
```

두번째 다른 표기법이다. 배열 arr의 포인터 타입은 int (*)[2] 로 표기할 수 있다.  
이 표기법이 유효한 것을 보여주기 위해 템플릿 함수 ArrayPoint의 템플릿 인자로 arr의 타입을 전달하고 정상작동 되는 것을 확인하였다.

```c++
int Func(int InArg)
{
    return InArg;
}

template<typename T>
void CallFunc(T InArg)
{
    cout << InArg(1) << endl;
}

void main()
{
    CallFunc<int (*)(int)>(Func); 
}
```

세번째 다른 표기법이다. 이번엔 함수를 포인터타입으로 쓰는 예제이다.  
Func의 포인터 타입은 int (*)(int) 로 표기할 수 있다.  
  
포인터 타입의 표기법이 단순히 원천타입 뒤에 간접연산자(*)를 붙여주는 것과 다르다는 것을 알 수 있다. 어떤 규칙으로 저렇게 표기하는 것일까?  

1. 원천타입의 객체를 선언한다.
(ex. int Func(int) {})
2. 객체 이름을 간접 연산자(*)로 바꾸어준다.  
(ex. int *(int) {})  
3. 간접연산자를 괄호로 감싸서 우선순위를 맞춘다.
(ex. int (*)(int) {})
4. 맨 끝에 세미콜론(;) 이나 대괄호({})를 없애준다.
(ex. int (*)(int))

이런 과정을 통하여 포인터 타입을 표기할 수 있다.  
우리가 흔히 하는 TYPE* 방식도 이런 과정을 통하여 포인터 타입으로 표기가 되는 것이다.

## 🔥 포인터와 주소 대응
  
> 포인터는 메모리의 주소를 나타내므로 0을 포함하는 자연수를 나타낸다.  
  
그렇지만 포인터에 덧셈, 뺄셈은 다르게 적용된다.  
정수형 타입처럼 +,- 연산이 되는 것이 아닌, 포인터 타입으로 +,- 연산이 된다.  
  
```c++
void main()
{
    char c;
    int i;
    double d;

    char* pC = &c;
    int* pI = &i;
    double* pD = &d;

    long addC = (pC + 1);
    long addI = (pI + 1);
    long addD = (pD + 1);
}
```

char, int, double 각각 다른 타입의 포인터에 + 1을 해보았다.  
정수형 타입처럼 연산이 된다면 각각의 주소에 1만큼 더한 주소가 나와야한다.  
하지만, 각각 다르게 더한 주소가 나오게 된다.  
  
addC는 1만큼 더한 주소, addI는 4만큼 더한 주소, addD는 8만큼 더한 주소가 나오게 된다.  
이는, 각각 포인터의 원천타입 바이트수만큼 더해진 것이다.  
  
즉, 포인터의 덧셈, 뺄셈 연산은 포인터의 원천타입의 크기의 배수만큼 증가 혹은 감소한다.  
(ex. double포인터 주소에 + 2를 하면 8바이트 * 2를 하므로 주소가 16만큼 증가한다.)  
  
## 🔥 포인터 연산자

> C++은 포인터를 통하여 메모리를 쉽게 다룰 수 있게 다양한 연산자를 제공한다.

- 간접 연산자 (*) 
    * 메모리의 주소를 통해서 간접적으로 메모리를 나타낸다.  
    * 포인터 객체 앞에 * 를 붙이면 포인터가 가리키는 주소의 메모리 블록에 접근할 수 있다.  
            
- 간접 멤버 연산자 (->)  
    * 포인터의 원천 타입이 클래스인 경우에만 클래스 멤버를 나타내기 위하여 사용한다.  
    * 포인터 객체 뒤에 -> 를 붙이면 클래스 객체의 멤버가 사용중인 메모리 부분에 접근할 수 있다.
    * 포인터가 아닌 클래스 객체에서 멤버에 접근하기 위해서는 직접 멤버 연산자 (.)을 사용한다.
      
- 증감 연산자 (++, --)
    * 포인터가 가리키는 주소를 포인터의 원천 타입 크기만큼 이동한다.
    * 위 파트에서 본 듯이 포인터 타입으로 덧셈, 뺄셈 연산을 하는 것이다.  
      
- 첨자 연산자 ([])
    * 포인터를 배열처럼 사용할 수 있다.
    * 첨자안에 넣는 인덱스만큼 포인터 타입으로 증감 연산을 한다.  
    즉, (인덱스 * 원천타입의 크기) 만큼 주소를 이동시키고 그 메모리에 접근한다.
