# 🚀 Chapter 1-2 : 부동소수점 타입

## 🔥 부동소수점 사용 이유

> 부동소수점이란 실수를 표현할 때 소수점의 위치를 고정하지 않는 것을 의미한다.  
  
왜 소수점의 위치를 고정하지 않을까?  
소수점의 위치를 고정하면 한정된 비트로 정수와 소수부분을 나누어 <u>표현할 수 있는 값의 범위가 작아지기 때문이다.</u>  
  
부동소수점은 소수점의 위치만 기억하면 되므로 비트를 아낄 수 있다.  
<u>즉, 같은 비트로 더 큰 수와 더 작은 수를 많이 표현하기 위해서 사용하기 위해서이다.</u>  

## 🔥 부동소수점 2진법 표현법

부동소수점을 2진법으로 표현하기 위해서는 실수를 2진법으로 표현할 줄 알아야 한다.  
  
- 계산 방법  
  * 정수의 2진법을 구할 때에는 정수를 2로 나누어 나머지를 구하며 2진법으로 변환하였다.
  * 소수점 이하의 2진법은 그와 반대로, 소수를 2로 곱해 정수를 구하며 2진법으로 변환하면 된다.  
  
예를 들어, 10진수 -9.6875를 2진법으로 표현해보겠다.  
  
![Fundamental-Cpp-BlogImages-2](https://user-images.githubusercontent.com/39856790/159156763-cfa8000f-90b3-4466-8437-9f8135c1686a.jpg)  
  
정수부분 9를 먼저 2진법으로 표현하면 1001이 된다.  
나머지 소수부분 0.6875를 2진법으로 표현하면 1011이 된다.

고로 -9.6875(10진수) -> -1001.1011(2진수) 가 된다. 

## 🔥 부동소수점 구조 

실수를 2진법으로 표현할 수 있게 되었지만, 이를 컴퓨터에서 어떻게 저장할까?

부동소수점 형식은 **부호부**, **지수부**, **가수부** 로 세가지로 비트들을 분배하여 표현 및 저장한다.

이 분배 형식은 여러가지가 있으며, 표준으로 많이 사용되는 형식인 IEEE754는 이렇게 분배된다.

|타입|부호부|지수부|가수부|total bit|
|:---:|:---:|:---:|:---:|:---:|
|float|1bit|8bit|23bit|32bit|
|double|1bit|11bit|52bit|64bit|  
  
예를 들어, IEEE754방식으로 float타입으로 -9.6875를 표현해보면  
\\(-9.6875 \\) (10진수) → \\(-1001.1011 \\) (2진수) -> \\(-1.0011011 * 2^3 \\) 로 나타낼 수 있다.  
\\(-1.0011011 * 2^3 \\) 에서의 부호부, 지수부, 가수부는 이렇게 표현된다.  
  
<!-- 부호부, 지수부, 가수부를 표현하는 이미지 삽입 필요 -->

<u>가수의 정수부는 언제나 1이 있다고 가정한다.</u>  
그렇기에 가수부는 정수부까지 고려하면 24비트의 정밀도를 가진다.  
  
지수부는 8bit이므로 256개의 다른 값을 나타낼 수 있다.  
그런데, 지수는 음수가 나올수도 있다.  
  
🔔   
얘를 들어, 10진수 0.625는 2진수로 변환하면 0.101이다.  
이를, float에서 저장하면 \\(1.01 * 2^-1 \\) 이 된다.
{: .notice}  
  
하지만, <u>지수부에는 부호비트가 없어 음수를 표현할수가 없다.</u>  
지수용 부호비트를 따로 만들면 계산이 복잡해지므로 지수부 8비트가 나타낼 수 있는 256개의 수를 반으로 나눠  
각각 음수와 양수를 나타내도록 하였는데, 이것이 <u>바이어스 표현법</u>이다.  
  
즉, 지수부에서의 음수는 <u>2의 보수법이 아닌 바이어스 표현법을 택한다.</u>    
그렇기에 지수는 \\(2^-127 ~ 2^128 \\) 의 범위로 나타낼 수 있다.  
  
위에서 가수부의 정수부는 언제나 1이라고 하였는데, 이러면 0을 표현할 수 없다.  
그래서 IEEE754는 지수부가 모두 0으로 채워지면 가수부의 정수부도 0으로 만들어버린다.  
그리고, 지수부가 모두 0이라면, -127이 아닌 -126으로 간주한다. (-127의 지수를 표현할 수 없기 때문에)  
그렇기에 지수부가 표현할 수 있는 최소는 -126이 되며,  
모든 지수부가 1로 채워진 128도 특별한 경우로 쓰이기 때문에,  
사실상 <u>IEEE754 float의 지수 표현 범위는 \\(2^-126 ~ 2^127 \\) </u>라고 할수 있다.

## 🔥 부동소수점 표현 한계

> 부동소수점 구조를 보면 float과 double등 부동소수점의 가수부 크기는 일정하다.  
그렇기에 지수가 어느 이상 커지면 오차가 생길 수 있다.  

IEEE754에서 float을 예로 들어보겠다.  
부호부 1bit, 지수부 8bit, 가수부 23bit를 써서 32bit(4byte)를 사용한다.  
  
그런데 지수가 가수부 bit수인 23을 넘으면 어떻게 될까?  
  
일단, float값이 0이고 지수를 가장 최대인 23으로 설정해보자.  
23은 바이어스 표현법으로 2진수로 변환하면 10010110이다.  
가수부의 정수부는 1이므로, float의 값은 \\(1 * 2^23 = 2^23 \\) 이 될 것이다.  
\\(2^23 \\)값을 계산하면 8388608이 된다.  
그래서 <u>float은 8388608 값 이상이 되면, 소수점 이하의 자리를 제대로 표현할 수 없게 된다.</u>  
  
```c++
/////////////////////////////////////
// Source

void main()
{
  float f[10];

  f[0] = 8388608.0;
  f[1] = 8388608.1;
  f[2] = 8388608.2;
  f[3] = 8388608.3;
  f[4] = 8388608.4;
  f[5] = 8388608.5;
  f[6] = 8388608.6;
  f[7] = 8388608.7;
  f[8] = 8388608.8;
  f[9] = 8388608.9;

  // 32자리 소수점까지 보이도록 설정
  // 설정한 이유는 설정하지 않으면 지수 표기법(E)형식으로 보이면 확인하기 어렵기 때문
  cout.precision(32);
  for (int i = 0; i < 10; i++)
  {
    cout << f[i] << endl;
  }
}

/////////////////////////////////////
// Result

8388608
8388608
8388608
8388608
8388608
8388608
8388609
8388609
8388609
8388609
```

결과가 이렇게 나온 이유는, <u>float은 8388608 이상 값에서 소수 부분을 표현할 수 없기 때문이다.</u>  
  
소수가 입력될 경우 가장 근사한 정수값을 선택하므로, 반올림을 하게 된다.  
<u>그러므로, 소수는 큰 값을 표현할 때 오차가 생기게 된다.</u>
  
지수가 23보다 클 때에는, 정수값에 오차가 점점 커지게 된다.  
지수가 24일 경우에는 짝수 정수만 표현이 가능하다.  
지수가 25일 경우에는 4의 배수 정수만 표현이 가능하다.  
지수가 26일 경우에는 8의 배수 정수만 표현이 가능하다.  
지수가 27일 경우에는 16의 배수 정수만 표현이 가능하다.
...  
  
<u>즉, 지수가 23이상이 되며, 점점 커진다면 표현가능한 정수의 격차는 2의 제곱씩 커지게 된다.</u>

## 🔥 사용시 주의점

위에서 봤듯 <u>첫번째로 주의해야 할 것은 큰 값을 다룰 때이다.</u>  
정확도를 요구하는 프로그램에서 부동소수점의 큰 값을 다룰 때에는 float을 사용하면 안된다.  
  
<u>float 대신에 어떤 플랫폼에서든 64비트를 유지하는 long long을 사용하도록 하자.</u>  
  
두번째로 주의해야 할 점은 부동소수점의 반올림이다.  
예를들어 0.4255를 소수점 세자리까지만 표현한다고 했을 때  
반올림을 한다면 0.426이 출력되어야 한다.  
  
하지만, 실제로는 0.425가 출력이 된다.  
왜냐하면, 0.4255를 부동소수점으로 표현하면 0.42549999...가 되기 때문이다.  
<u>그렇기에 반올림을 정확하게 수행하고자 한다면, 반올림 경계를 넘을만큼 작은 값을 더해주어야 한다.</u>

## 🔥 무한 & NaN

> 부동소수점 표현 한계 부분에서 지수부의 모든 비트가 1인 경우는 특별한 경우에 쓰인다고 하였다. 그 특별한 경우에 대해서 알아보자.
  
**무한 (INF)**  
  
무한을 표현하는 방법은 지수부의 비트가 모두 1인 상태에서 <u>가수부의 비트가 모두 0이어야 한다.</u>
  
무한은 양의 무한, 음의 무한으로 나뉘어진다.  
이를 나누는 기준은 부호비트가 결정한다.  
  
**NaN**  
  
NaN은 잘못된 입력으로 인해 계산을 할 수 없음을 나타낸다.  
  
NaN을 표현하는 방법은 지수부의 비트가 모두 1인 상태에서 <u>가수부의 비트가 하나라도 1이면 된다.</u>
