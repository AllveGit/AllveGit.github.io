# 🚀 Chapter 5-5 : STL Vector

> 이전 파트에서 설명했던 가변 크기 배열은 거의 사용하지 않으며,  
사용할 일이 있어도 대신 STL Vector를 채용한다.  

STL Vector는 배열의 장점은 유지하면서 단점들은 덮을 수 있는 자료구조이다.

## 🔥 배열과 vector

실제 STL Vector의 사용방식을 보면 배열과 거의 일치하다.  
오히려, vector에 더 많은 기능과 장점이 있기에 상위호환으로 볼 수도 있다.  
즉, 배열 대신에 vector만 써도 충분할수도 있다.

하지만, <u>vector는 기능적으로 배열을 대체할 수 있겠지만, 완벽하게는 대체하지 못한다.</u>  

```c++
void Func(int (&arg)[3])
{
  // ...
}

void main()
{
  vector<int> v(3);
  Func(v); // Compile Error!
}
```

예제 코드에서는 <u>vector가 배열 타입으로 인자를 전달할 수 없다</u>는 것을 보여준다.  

이번에는, 배열 타입이 아닌 포인터 타입으로 전달해보겠다.

```c++
typedef int (*PARR)[3];

void Func(PARR pArg)
{
  
}
```
