# 🚀 Chapter 2-3 : 정적 변수

## 🔥 정적 전역 변수

> 정적 전역 변수는 전역 변수 앞에 static을 붙인 변수이다.
  
<u>선언한 소스파일(cpp)에서만 접근 가능</u>한 점을 빼고 전역 변수와 다른 것이 없다.

## 🔥 정적 멤버 변수

> 정적 멤버 변수는 클래스 멤버 변수 앞에 static을 붙인 변수이다.

그러나 정적 멤버변수는 주의해야 할 점이 있다.
  
❗  
클래스 정의에서 정적 멤버 변수를 정의하였다고 해서 
실제로 정적 멤버 변수가 정의된 것이 아니다.
{: .notice--warning}
  
```c++
/////////////////////////////////////
// A.h

class Test
{
public:
  static int s_Val;
}

/////////////////////////////////////
// Main.cpp

#include "A.h"

int Test::s_Val = 1; // 정의

void main()
{
  cout << Test::s_Val << endl;
}
```

클래스 정의에 선언된 정적 멤버 변수 s_Val은 단지 클래스 설계도에 포함되는 것뿐이다.
그렇기에 s_Val은 <u>메모리 영역을 할당받기 위해 따로 정의</u>를 해주어야 한다.
  
정적 멤버 변수는 <u>전역 변수와 똑같지만 클래스 범위에서만 접근이 가능할 뿐</u>이다.
그렇기에 클래스 객체의 영향을 받지 않고 클래스 객체가 점유하는 메모리 영역과는 상관이 없다.


## 🔥 정적 지역 변수

> 정적 지역 변수는 함수 안의 지역 변수 앞에 static을 붙인 변수이다.
  
정적 지역 변수도 마찬가지로 <u>전역 변수와 똑같지만 함수 안에서만 접근이 가능한 변수</u>이다.    
그렇기에 함수가 시작될 때 생성되는 것이 아닌, <u>프로세스가 시작할 때 이미 생성 및 초기화가 되어 있는</u> 것이다.  
  
하지만, 정적 지역 변수가 전역 변수와 큰 다른 점이 하나 있다.  
  
```c++
int GetNum()
{
  return 7;
}

void FuncTest()
{
  static int s_Val = GetNum();

  cout << s_Val << endl;
}
```
  
만약 함수 FuncTest를 호출하게 되면 s_Val은 어떻게 초기화될까?  
s_Val은 프로세스가 시작될 때 생성되고 함수로 초기화를 하니, BSS 영역에서 먼저 0으로 초기화 시켜줄 것은 알겠지만, 정적 지역 변수 s_Val의 <u>초기화 시점은 함수 FuncTest가 호출될 때</u>이다.  
  
s_Val은 함수 FuncTest가 최초로 호출되었을 때만 7로 초기화하게 된다.
C++은 어떻게 FuncTest가 최초 호출되었을때만 초기화를 시키는걸까?

<!-- 어셈블리 이미지 삽입 필요 -->

그것은 바로 <u>컴파일러 자체가 따로 전역 변수를 만들어 초기화 여부를 기록</u>한다. (VC++ 기준)
전역 변수의 bit를 사용하여 정적 지역 변수 하나의 초기화 여부를 대응하며, bit가 부족해지면 새로운 전역 변수를 또 생성한다.

함수 초기화 방식이 아닌 초기화를 한다면 일반 전역 변수와 똑같이 초기화한다.

❗  
정적 지역 변수를 정의할 때 클래스 타입으로 정의를 한다면,  
클래스 생성자를 통해 초기화를 하므로 똑같이 함수 초기화 방식을 따른다.
{: .notice--warning}
