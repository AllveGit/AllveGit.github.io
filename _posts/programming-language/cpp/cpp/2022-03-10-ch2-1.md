# 🚀 Chapter 2-1 : 선언과 정의의 구분

#### 🔥 정의
***

> C++에서의 정의는 객체를 실제로 만들겠다는 것을 의미한다.
즉, 만들 객체를 위한 메모리 영역을 확보하는 것을 말한다.
  
<br>
```c++
class Test
{
    
};

int a; //(1)

int main()
{
    int b; //(2)
    Test c; //(3)
}

int Add(int InA, int InB) //(4)
{
    return InA + InB;
}
```

(1)~(4)는 모두 메모리 영역을 확보하기 위해 정의를 한 구문들이다.

(4)번 케이스, **함수도 메모리 영역을 차지한다.** 
함수 본체가 있을 경우, 함수 본체를 어셈블리로 변환하고, 그 어셈블리가 코드 영역에 저장이 된다.
따라서 함수도 일반 객체처럼 메모리 공간을 차지한다.

🔔 정의란 어떤 대상이 실제로 존재할 수 있도록 메모리 영역을 만들어주는 작업이다.
{: .notice}

<br> 하지만, 클래스의 경우 정의 개념이 달라진다.

> 클래스에서의 정의는 클래스가 어떻게 구성되어 있는지 알 수 있는 설계도를 의미한다.

기존 다른 정의들과 다른 점은, 클래스 설계도는 프로그램이 사용하는 메모리가 아닌,  
컴파일러가 사용하는 메모리에 저장이 된다.

<br>
```c++
class Test
{
public:
    int m_Value;

    int MFunc(int arg)
    {
        return arg;
    }
};
```
MFunc와 같은 클래스 멤버 함수는 프로그램이 시작할 때,  
다른 함수들과 마찬가지로 어셈블리로 변환해 코드 영역의 메모리에 저장된다. 

<br>

#### 🔥 선언
***

> 선언이란 실제 객체가 존재하고 있음을 알리는 것을 의미한다.

그런데 정의도 객체를 정의하는 순간 해당 객체가 알려지게 된다.

선언과 정의는 둘이 다른 것이 아닌 정의가 선언을 포함한다.
그렇기에 정의를 하면 선언도 된다는 것이다.

그런데도 선언이 굳이 있는 이유는 **선언이 필요할 때가 있기 때문** 이다.

<br>
```c++
/////////////////////////////////////
// A.cpp

int g_Val;
int Func()
{
    return 1;
}

/////////////////////////////////////
// Main.cpp

void main()
{
    g_Val = Func(); // (1) Compile Error!
}
```

예제는 A.cpp에서 전역으로 g_Val과 Func를 정의 및 선언하였다.  
(1) Compile Error가 나는 이유는 이 선언 범위가 A.cpp에만 해당된다는 것이다.

그렇기에 g_Val과 Func라는 전역함수가 있다는 선언을 해주어야 Main.cpp에서도 존재를 알 수 있다.  

❗ 정의는 선언을 포함하므로 Main.cpp에서도 정의를 해주면 되지 않을까?  
그렇게 되면 링크 에러가 발생한다.  
둘 중 어느 것을 선택해야 하는 지 모호해지기 때문에 중복 정의는 허용되지 않는다.
{: .notice-warning}

<br>
```c++
/////////////////////////////////////
// A.cpp

int g_Val;
int Func()
{
    return 1;
}

/////////////////////////////////////
// Main.cpp

extern int g_Val;   // (1)
int Func();         // (2)

void main()
{
    g_Val = Func();
}
```

(1), (2) 같이 각각 객체가 존재한다는 선언을 해주면 컴파일 에러가 없어지게 된다.
