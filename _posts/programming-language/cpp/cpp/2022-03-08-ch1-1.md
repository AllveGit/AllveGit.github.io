# 🚀 Chapter 1-1 : 정수 타입

## 🔥 부호 없는 정수 (unsigned type)

> 음수가 없는 0을 포함한 자연수를 뜻한다.
    
컴퓨터에서는 비트마다 0 혹은 1의 값을 가질 수 있다.
그렇기에 정수를 2진법으로 표현한다.  
(ex. 3 -> 11)

🔔  
모든 비트가 0인상태는 숫자 0과 대응이 된다.
{: .notice}
  
## 🔥 부호 있는 정수 (signed type)

> 부호 없는 정수와 달리 음수를 표현할 수 있는 자연수를 뜻한다.
  
부호 있는 정수는 음수도 2진법으로 표현해야 한다.  
하지만, 컴퓨터에서의 음수 표현은 까다롭다, 2진값으로 **+**, **-** 의 부호를 표시해야 하기 때문이다.

그렇다면 <u>첫번째 비트를 부호 비트</u>로 사용하면 될 것 같다.
예를 들어, 4비트만 사용해보겠다.
  
|정수 (10진수)|컴퓨터 (2진수)|
|:---:|:---:|
|7|0111|
|6|0110|
|5|0101|
|4|0100|
|3|0011|
|2|0010|
|1|0001|
|0|0000|
|-0|1000|
|-1|1001|
|-2|1010|
|-3|1011|
|-4|1100|
|-5|1101|
|-6|1110|
|-7|1111|
    
**첫번째 비트를 부호비트로 설정하는 방식은 단점이 많다.**
- 음수를 표현하니 표현할 수 있는 최댓값이 반토막이 났다. 
- 0과 -0 둘 다 같은 값이지만 두개의 공간을 차지한다.
- 가장 큰 단점은 음수 계산을 위한 논리회로, 즉 누산기를 설계하기가 매우 복잡해진다.
  
그래서 음수 표현을 할 <u>2의 보수법</u> 이라는 새로운 방식이 도입된다.

## 🔥 1의 보수법, 2의 보수법

> 1의 보수란 어떤 2진수에서 각 비트를 반전시킨 것을 의미한다. (ex. 0011 -> 1100)  
2의 보수는 1의 보수값에서 1을 더한 값을 의미한다. (ex. 0011 -> 1101)
  
2의 보수법을 어떻게 사용한다는 것인가? 바로, <u>2의 보수값을 부호가 반대인 수로 사용한다는 것</u>이다.   
(ex. 3의 음수는 (3 -> 0011 -> 1101)를 통해 1101이 된다)
  
|정수 (10진수)|컴퓨터 (2의 보수)|
|:---:|:---:|
|7|0111|
|6|0110|
|5|0101|
|4|0100|
|3|0011|
|2|0010|
|1|0001|
|0|0000|
|-1|1111|
|-2|1110|
|-3|1101|
|-4|1100|
|-5|1011|
|-6|1010|
|-7|1001|
|-8|1000|
  
그 전의 표와 비교하면 달라진 점이 눈에 띈다.
- 0이 하나의 공간만 차지한다.
- 음수는 부호비트를 제외하고 -1을 가장 큰 수로 가정하며, 점점 작아진다.

<br> 2의 보수법을 사용하면 무슨 장점이 있을까?  
0이 하나의 공간만 차지하는 것? <u>가장 큰 것은 계산이 쉬워진다는 것이다.</u>

🔔   
4 + (-3)을 2의 보수법으로 계산하면 0100 + 1101 = (1)0001이 되며,  
맨 왼쪽의 넘치는 비트를 제거하면 0001 -> 1이 나오게 된다.  
이런식으로 덧셈회로만 이용해도 양수 음수 상관없이 계산을 할 수 있게된다.
{: .notice}
  
## 🔥 최솟값, 최댓값

> 4byte(32bit) 정수를 예로 들어보겠다.
  
최솟값 : <u>1을 설정하고 31비트를 왼쪽으로 이동</u>시키면 된다.
```
int minInt = 1;
minInt = minInt << 31; 
```

<br>

최댓값 : <u>최솟값을 반전</u>시키면 된다.
```
int maxInt = 1;
maxInt = maxInt << 31; 
maxInt = ~maxInt;
```  
  
## 🔥 비트 이동 연산자 (shift operator)

> 비트 이동 연산자는 비트 상태를 변화시킬 수 있는 연산자를 의미한다.  
  
비트 이동을 할때에는 부호에 관련해서 주의해야 할 점이 있다.
  
```
char c = 1;

c = c << 7; (1)
c = c >> 7; (2)
```  
  
예제를 살펴보자,   
char는 1byte(8bit)이므로 1을 설정하면 00000001이 된다.  

(1)에서는 7bit만큼 왼쪽으로 이동했으므로 00000001 -> 10000000이 됐을 것이다.  
(2)에서는 7bit만큼 오른쪽으로 이동했으므로 10000000 -> 00000001이 됐을까?
  
틀렸다, (2)의 결과는 <u>11111111</u>이 된다.  
이런 결과가 나온 이유는 비트 이동 연산자의 <u>부호 유지 정책</u>때문이다.  

🔔 부호 유지 정책이란?   
비트 이동을 했을 때 새로운 비트가 추가 될 때 부호를 잃어버리지 않고 유지하기 위한 정책이다.  
{: .notice}

비트 이동의 규칙을 알아보자.

- "<< N" : 비트 상태를 N만큼 왼쪽으로 이동한다. (한 번 이동할 때마다 2를 곱한다.)  
가장 왼쪽의 넘치는 N개의 비트는 버림, 오른쪽에 새로 생기는 N개의 비트들은 모두 0으로 삽입  
  
- ">> N" : 비트 상태를 N만큼 오른쪽으로 이동한다. (한 번 이동할 때마다 2를 나눈다.)  
가장 오른쪽의 넘치는 N개의 비트는 버림, 왼쪽에 새로 생기는 N개의 비트들은  
**<u>부호 비트가 존재할 경우 부호 비트로 채우고, 존재하지 않는다면 0으로 채운다.</u>**

<br> 이 규칙에 따라 10000000 에서 7bit 오른쪽으로 이동하면,  
부호 비트가 1이니 계속 새 비트가 1로 채워지게 되므로 11111111 이라는 결과가 나오게 된다.

## 🔥 데이터 모델

> 데이터 모델은 각 타입의 크기가 어떻게 결정되는지 나타내는 걸 의미한다.
  
|data model|char|short|int|long|long long|pointer|
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
|ILP32|8bit|16bit|32bit|32bit|64bit|32bit|
|ILP64|8bit|16bit|64bit|64bit|64bit|64bit|
|LP64|8bit|16bit|32bit|64bit|64bit|64bit|
|LLP64|8bit|16bit|32bit|32bit|64bit|64bit|
  
❗  
타입에 따른 크기는 항상 일정하지 않다.  
왜냐하면, 플랫폼에 따라 데이터 모델이 다르기 때문이다.   
데이터 모델은 플랫폼에 종속적이다.  
플랫폼에 따라 타입의 크기가 다를 수 있다는 점을 명심하자!
